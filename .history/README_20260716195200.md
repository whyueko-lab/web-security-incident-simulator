# Web Security Incident Simulator

Web Security Incident Simulator adalah aplikasi berbasis **CodeIgniter 4** yang dikembangkan untuk mensimulasikan berbagai insiden keamanan aplikasi web dan proses penanganannya. Aplikasi ini dibuat sebagai media pembelajaran mengenai keamanan siber, deteksi serangan, pencatatan insiden, serta pemulihan sistem berdasarkan konsep **Incident Response**, **Business Continuity Plan (BCP)**, dan **Disaster Recovery Plan (DRP)**.

---

## Features

- 🔐 Login Authentication
- 💉 SQL Injection Simulation
- 🔓 Broken Object Level Authorization (BOLA) Simulation
- 🔒 Ransomware Simulation
- 💾 Backup & Disaster Recovery
- 📋 Attack Logs (Security Operation Center Dashboard)
- 📊 Dashboard Monitoring
- 🚨 Security Incident Detection
- ♻ Recovery Process Simulation

---

## Simulation Modules

### SQL Injection
Simulasi serangan SQL Injection menggunakan mode **Vulnerable** dan **Secure** untuk menunjukkan bagaimana input yang tidak divalidasi dapat dieksploitasi serta bagaimana penggunaan Prepared Statement dapat mencegah serangan.

### Broken Object Level Authorization (BOLA)
Simulasi akses rekam medis pasien menggunakan manipulasi ID pada URL. Modul ini menunjukkan perbedaan antara sistem yang tidak memiliki kontrol otorisasi dengan sistem yang telah menerapkan validasi hak akses.

### Ransomware
Simulasi serangan ransomware yang menyebabkan database terenkripsi, layanan berhenti beroperasi, serta proses pemulihan menggunakan backup.

### Backup & Recovery
Simulasi proses pembuatan backup, restore database, serta implementasi Disaster Recovery Plan (DRP).

### Attack Logs
Seluruh aktivitas simulasi dicatat ke dalam log sehingga administrator dapat melakukan monitoring seperti pada Security Operations Center (SOC).

## Installation

Clone repository

```bash
git clone https://github.com/whyueko-lab/web-security-incident-simulator.git
```

Masuk ke folder project

```bash
cd web-security-incident-simulator
```

Install dependency

```bash
composer install
```

Copy file environment

```bash
cp env .env
```

Sesuaikan konfigurasi database pada file `.env`.

Jalankan server

```bash
php spark serve
```

Buka browser

```
http://localhost:8080
```
